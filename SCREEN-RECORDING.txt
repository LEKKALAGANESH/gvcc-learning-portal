====================================================================
 SCREEN RECORDING SCRIPT - GVCC LEARNING PORTAL
====================================================================

One continuous script. Each section has SHOW (what to do on screen)
and SAY (read it aloud). Go top to bottom. Target ~5-6 min, 1080p,
record with OBS or Win+G.

BEFORE YOU START:
  - Run: npm run db  ->  npm run dev  ->  open localhost:3000
  - Log OUT.
  - Have login ready to paste: demo@gvcc.dev / password123
  - Watch a few seconds of one video first so the Continue-watching
    row is populated.
  - Open VS Code in a second window for the code part.


--------------------------------------------------------------------
SECTION 1 - INTRO
--------------------------------------------------------------------

SHOW: The app open on the /login screen (logged out).

SAY:
"Hi - this is my GVCC Learning Portal. It's a student portal where
you watch lessons and drop multiple timestamped bookmarks on any
video, then resume from any bookmark or from wherever you left off.
It's built with Next.js 15 and React 19, TypeScript in strict mode,
Prisma with SQLite, JWT auth in an httpOnly cookie, and Zod
validating every input - one codebase for the UI and the API."


--------------------------------------------------------------------
SECTION 2 - AUTHENTICATION
--------------------------------------------------------------------

SHOW: Click Sign up, create a throwaway account
(you@test.dev / password123), land in the portal. Then log out and
log back in with demo@gvcc.dev / password123.

SAY:
"I'll start signed out. I can sign up for a new account here... and
I'm in. Now let me log out and sign back in with the demo account.
Sessions are a signed JWT in an httpOnly cookie, and middleware
redirects any signed-out user straight to the login page."


--------------------------------------------------------------------
SECTION 3 - THE LIBRARY / PORTAL
--------------------------------------------------------------------

SHOW: The library grid - point at a card's category, duration,
thumbnail, and its progress bar. Point at the Continue watching and
Recently watched rows. Type "design" in search to filter live.

SAY:
"This is the library. Each lesson card shows its category, duration,
thumbnail, and a watch-progress bar. Up top there's a
Continue-watching row and a Recently-watched row, and search filters
the library live - let me type 'design'. All of this is
database-backed and per user."


--------------------------------------------------------------------
SECTION 4 - VIDEO PLAYER + BOOKMARKS (THE CORE)
--------------------------------------------------------------------

SHOW: Open "Introduction to Motion Design". Play a few seconds,
press B, label it "Intro". Scrub ahead, add a 2nd bookmark
("Key concept"), scrub again, add a 3rd. Show the panel listing all
three sorted by time, and the timeline ticks under the player. Then
click the bookmark at ~10:45 and let the player visibly jump there.
Press [ and ] to hop prev/next.

SAY:
"Opening a lesson. I'll play a few seconds and press B to bookmark
this moment - I'll label it 'Intro'. Now I'll scrub ahead and add a
second bookmark, and a third. So: multiple bookmarks per video, all
listed and sorted by timestamp, with clickable ticks on the timeline
under the player. Here's the key requirement - I click the bookmark
at 10:45, and the player jumps straight to 10:45, not back to zero.
And the bracket keys jump to the previous and next bookmark."


--------------------------------------------------------------------
SECTION 5 - EDIT, DELETE & SHARE LINKS
--------------------------------------------------------------------

SHOW: Rename a bookmark inline. Delete one. Click a bookmark's
copy-link action, paste the URL in the address bar showing ...?t=645
deep-linking to that second.

SAY:
"I can rename a bookmark inline, delete one, and copy a share link -
this URL deep-links straight to that exact second. Every one of
these mutations is Zod-validated and ownership-checked on the
server."


--------------------------------------------------------------------
SECTION 6 - PERSISTENCE
--------------------------------------------------------------------

SHOW: Refresh the page - the bookmarks and resume position are all
still there.

SAY:
"If I refresh - everything's still here: the bookmarks and my resume
position, because it all lives in the database, scoped to my
account."


--------------------------------------------------------------------
SECTION 7 - SCREENSHOT PROTECTION
--------------------------------------------------------------------

SHOW: On the watch page, Alt-Tab / click another window -> video
blurs; come back -> un-blurs. Press PrintScreen -> warning toast
appears. Right-click the video and try to drag it -> both blocked.

SAY:
"For screenshot protection I want to be honest about the platform: a
browser genuinely cannot block an OS-level screenshot - that capture
happens outside the page. So I ship the strongest practical web
deterrents and document the limit. Watch - when I switch windows,
the protected video blurs immediately, which is exactly when a
capture tool grabs a frame. PrintScreen clears the clipboard and
flashes a warning. Right-click, drag, and save shortcuts are
blocked, downloads are off, and there's HTTP hardening against
embedding. For true prevention I document the native paths in the
README: Android FLAG_SECURE, iOS capture notifications, and
Electron's setContentProtection."


--------------------------------------------------------------------
SECTION 8 - RESPONSIVE
--------------------------------------------------------------------

SHOW: Open DevTools device toolbar (Ctrl+Shift+M), cycle
375 -> 768 -> 1440. Show the grid and watch page reflow with no
horizontal scroll.

SAY:
"It's responsive - here it is at mobile, tablet, and desktop widths;
the grid and player reflow cleanly with 44-pixel touch targets. I
verified 320 through 1440."


--------------------------------------------------------------------
SECTION 9 - CODE, API & TESTS
--------------------------------------------------------------------

SHOW (VS Code): prisma/schema.prisma - the User, Video, Bookmark,
Progress models. A route in src/app/api/bookmarks/ - point at Zod
validation + the userId ownership check. src/components/ - note
reusable VideoCard / BookmarkPanel / WatchExperience. Run npm test
in the terminal, show Vitest green.

SAY:
"Quickly under the hood: the Prisma schema models User, Video,
Bookmark, and Progress. The API routes validate with Zod and scope
every row to the authenticated user. Components are reusable and
data-driven. And there's a unit-test suite - running it now, all
passing."


--------------------------------------------------------------------
SECTION 10 - CLOSE
--------------------------------------------------------------------

SHOW: Back on the library, or a glance at the README.

SAY:
"That's the GVCC Learning Portal - every required feature, all the
bonus features, and honest, documented screenshot protection. To
deploy, I'd just point DATABASE_URL at Postgres. Thanks for
watching."
